# Deep Interview Context Snapshot

- Task statement: 在 `com.bytefuture.easy.poster.element.chart` 包下增加一个饼图元素。
- Desired outcome: 形成可执行规格，重点澄清输入配置项、图例、颜色策略。
- Stated solution: 新增饼图绘制能力，风格与现有 Java 8 图像绘制工具一致。
- Probable intent hypothesis: 希望扩展现有 chart 能力，在海报/图片中支持占比型数据展示，并与现有 BarChartElement API 风格保持一致。
- Known facts/evidence:
  - chart 包目前已有 `BarChartElement` 与 `BarChartSeries`。
  - `EasyPoster` 当前提供 `addBarChartElement(int width, int height)`，暂无饼图入口。
  - `BarChartElement` 已有图例开关与样式：`showLegend`、`legendFontSize`、`legendItemGap`、`legendMarkerSize`。
  - `BarChartElement` 已有默认色板 `DEFAULT_PALETTE`，并优先使用 series 自定义颜色。
  - 图表测试位于 `src/test/java/com/bytefuture/easy/poster/ui/chart/BarChartBasicTest.java`。
- Constraints:
  - Java 8。
  - 需要遵循当前 brownfield API/命名模式。
  - 本轮是 requirements clarifying，不直接实现。
- Unknowns/open questions:
  - 饼图的数据模型是单序列切片，还是支持多组/环形等扩展。
  - 图例是否必须显示百分比/数值/名称，位置是否可配置。
  - 颜色是逐项指定、统一色板回退，还是需要字符串色值 API。
  - 标题、标签、起始角度、是否显示扇区内文本是否属于首版范围。
- Decision-boundary unknowns:
  - OMX 可否直接复用 BarChart 的 legend/color 默认值与命名风格。
  - OMX 可否只做首版基础饼图，不包含环形/3D/动画/交互。
- Likely codebase touchpoints:
  - `src/main/java/com/bytefuture/easy/poster/element/chart/`
  - `src/main/java/com/bytefuture/easy/poster/EasyPoster.java`
  - `src/test/java/com/bytefuture/easy/poster/ui/chart/`
