# easy-poster v0.1.1 版本更新说明

> 一个简单的 Java 海报制作工具，用代码绘制图片、矩形、二维码、文字，生成你想要的海报。

---

## 一、文本元素 · 纵向绘制

古人写字，是从右往左、从上往下竖排书写的。现在你也可以在 easy-poster 里还原这种排版。

一行代码开启竖排模式：

```java
TextElement.of("春眠不觉晓处处闻啼鸟夜来风雨声花落知多少")
    .vertical()            // 切换竖排书写
    .columnRightToLeft()   // 列方向：从右到左（传统中文排版）
    .maxVerticalWidth(400) // 列高上限
    .setFontSize(28)
    .setColor(Color.BLACK)
    .setPosition(RelativePosition.of(Direction.TOP_LEFT, Margin.of(24)))
```

**效果示意：**

```
  列4  列3  列2  列1
  ↓    ↓    ↓   ↓
  花   夜   处   春
  落   来   处   眠
  知   风   闻   不
  多   雨   啼   觉
  少   声   鸟   晓   
```

### 核心能力

| 特性 | 说明 |
|------|------|
| **列方向** | `columnRightToLeft()` 传统右起；`columnLeftToRight()` 左起 |
| **列高控制** | `maxVerticalWidth(int)` 控制单列最大高度 |
| **列内对齐** | `setVerticalAlign(VerticalAlign)` — TOP / CENTER / BOTTOM |
| **列间距** | `setColumnSpacing(int)` 列与列之间的间距 |
| **行高** | `setLineHeight(int)` 控制列宽（竖排中"行高"等于列宽） |

### 标点避头尾

竖排不是简单地把横排文字旋转 90 度。中文排版有**标点避头尾**规则——句号不能出现在列首，逗号不能出现在列尾。

easy-poster 内置了 `PunctuationAdjuster`，自动处理这些情况：

- **避头**：句号、叹号、问号、右括号等不放在列首，挤回前一列末尾
- **避尾**：逗号、左括号等不放在列尾，推到下一列开头

你不需要手动处理，引擎自动完成。

## 二、文本元素 · 渐变色

文字不再是单调的纯色。给文本加上渐变，一行搞定：

```java
TextElement.of(TextSpan.of("Gradient "), TextSpan.of("Color"))
    .setFontStyle(Font.BOLD)
    .setFontSize(42)
    .setGradient(Gradient.of(
        new Color[]{new Color(255, 96, 72), new Color(86, 92, 255)},
        GradientDirection.LEFT_RIGHT))
    .setPosition(RelativePosition.of(Direction.TOP_LEFT, Margin.of(24)))
```

**效果示意：**

```
  Gradient Color
  ═══════════════════
  红──────────────→蓝
  (255,96,72)   (86,92,255)
```

### 四种渐变方向

| 方向 | 枚举值 | 效果 |
|------|--------|------|
| 上→下 | `TOP_BOTTOM` | 垂直渐变 |
| 左→右 | `LEFT_RIGHT` | 水平渐变 |
| 左上→右下 | `TOP_LEFT_RIGHT_BOTTOM` | 对角渐变 |
| 右上→左下 | `TOP_RIGHT_LEFT_BOTTOM` | 反对角渐变 |

### 渐变 + 富文本混搭

渐变色是块级设置，作用于整个文本元素。但如果某个 TextSpan 设了自己的颜色，它会**覆盖**渐变，保持自己的纯色：

```java
TextElement.of(
    TextSpan.of("渐变部分"),
    TextSpan.of("纯色部分").setColor(Color.RED))  // span 级颜色覆盖渐变
    .setGradient(Gradient.of(...))
```

这种「块级渐变 + span 级颜色优先」的机制，让你在一行文字里同时拥有渐变和纯色。

### 多色渐变

渐变不止双色，可以传任意数量的颜色数组，还支持自定义 fractions 控制各颜色的分布比例：

```java
Gradient.of(
    new Color[]{Color.RED, Color.YELLOW, Color.GREEN, Color.BLUE},
    GradientDirection.TOP_BOTTOM)
    .setFractions(new float[]{0.0f, 0.3f, 0.6f, 1.0f})
```

## 三、文本元素 · 最大行数与文本缩略

长文本自动换行后，你可能不希望它无限延伸。现在可以用 `maxLines` 控制最大行数，配合 `textOverflow` 决定超出的部分怎么处理：

```java
TextElement.of("这是一段超长文本的测试，验证自动换行并进行缩略...")
    .setFontSize(20)
    .setMaxTextWidth(220)   // 先启用自动换行
    .setMaxLines(2)         // 最多显示 2 行
    .setTextOverflow(TextOverflow.ELLIPSIS)  // 超出部分加省略号
```

**ELLIPSIS 模式效果：**

```
这是一段超长文本的测试，验证
自动换行并进行缩略。这是一...
```

**CLIP 模式效果（直接截断，不加省略号）：**

```
这是一段超长文本的测试，验证
自动换行并进行缩略。这是一段超
```

> **注意：** `maxLines` 需配合 `maxTextWidth`（自动换行）使用，未开启自动换行时该参数不生效。

### 省略号的智能选择

ELLIPSIS 模式不是简单硬加 "..."，引擎会根据剩余宽度智能选择最长的省略号表示：
- 宽度够 → "..."
- 宽度紧 → ".."
- 宽度极窄 → "."

## 四、移除全局配置 debug 参数

旧版本中 `Config` 有一个 `debug` 参数，开启后会给每个元素画黑色边框，用于排查布局问题。这个功能在实际使用中价值有限，且增加了代码维护负担。

**移除内容：**
- `Config.debug` 字段及其 getter/setter
- `AbstractElement.debug()` 方法及渲染调用
- 旧版 `TextElement` 中绘制基线位置的逻辑

如果你之前使用了 `config.setDebug(true)`，升级后需要删除该调用。调试布局问题建议通过调整元素间距或使用矩形元素辅助定位。

## 五、TextElement 架构升级

这是本次更新中**最大的变化**。

### 旧版 TextElement 的问题

旧版 TextElement 是一个单体类，所有逻辑揉在一起：

- 只支持纯文本（单个 String），不支持富文本
- 字体、颜色、行高等属性全部散落在元素上
- 文本拆分、布局计算、渲染绘制全在一个方法里
- 418 行的拆分器 `TextSplitterSimpleImpl` 独占一个包

### 新版 TextElement 的架构

新版采用**流水线（Pipeline）架构**，将文本处理拆为三个独立阶段：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  样式解析    │ →  │  布局计算    │ →  │  渲染绘制    │
│ TextStyle   │    │ LayoutEngine │    │ TextRenderer │
│ Resolver    │    │              │    │              │
└─────────────┘    └─────────────┘    └─────────────┘
```

**样式解析** — `TextStyleResolver`

将 TextSpan 的 span 级样式、TextBlockStyle 的块级样式、Config 的全局样式三层级联，解析为统一的 `ResolvedTextStyle`。这个过程确保了优先级：span > block > config。

**布局计算** — `TextLayoutEngine` / `VerticalTextLayoutEngine`

横排和竖排各自有独立的布局引擎，计算每一行/列的确切位置、字符偏移、行间距。

**渲染绘制** — `HorizontalTextRenderer` / `VerticalTextRenderer`

拿到布局结果后，绘制到画布上。渐变色、span 级颜色覆盖、下划线、删除线、字间距、背景色——全部在这个阶段处理。

### 旧版到新版 API 对照

旧版和新版的核心用法几乎没有变化，迁移成本极低：

| 旧版 | 新版 | 说明 |
|------|------|------|
| `TextElement.of("text")` | `TextElement.of("text")` | 不变 |
| `.setFontName(...)` | `.setFontName(...)` | 不变，委托给 blockStyle |
| `.setFontSize(...)` | `.setFontSize(...)` | 不变 |
| `.setColor(...)` | `.setColor(...)` | 不变 |
| 不支持 | `TextElement.of(TextSpan...)` | **新增**：富文本 |
| 不支持 | `.vertical()` | **新增**：竖排 |
| 不支持 | `.setGradient(...)` | **新增**：渐变 |
| 不支持 | `.setMaxLines(...)` | **新增**：行数限制 |
| 不支持 | `.setTextOverflow(...)` | **新增**：文本缩略 |
| 不支持 | `.setLetterSpacing(...)` | **新增**：字间距 |
| 不支持 | `.setUnderline(...)` | **新增**：下划线 |
| 不支持 | `.setStrikeThrough(...)` | **新增**：删除线 |

### 删除的旧代码

以下旧版代码已全部移除：

- `element.basic.TextElement`（旧版单体类）
- `element.basic.split` 包（ITextSplitter、SplitTextInfo、TextSplitRequest、TextSplitResult、TextSplitterSimpleImpl）
- `element.v2` 包整体升级为 `element.basic`，不再有 v2 子包

## 升级指南

**1. Maven 依赖**

```xml
<dependency>
    <groupId>cn.com.bytefuture</groupId>
    <artifactId>easy-poster</artifactId>
    <version>0.1.1</version>
</dependency>
```

**2. 迁移步骤**

- 删除所有 `config.setDebug(true)` 调用
- 如果引用了旧版 `element.basic.split` 包的类，移除相关 import
- 旧版 `TextElement.of("text")` 的用法完全兼容，无需修改
- 新功能按需使用，不影响已有代码

**3. 包路径变更**

旧版 v2 包路径 → 新版正式包路径：

| 旧路径 | 新路径 |
|--------|--------|
| `element.v2.text.layout.*` | `element.basic.text.layout.*` |
| `element.v2.text.pipeline.*` | `element.basic.text.pipeline.*` |
| `element.v2.text.render.*` | `element.basic.text.render.*` |
| `element.v2.text.style.*` | `element.basic.text.style.*` |

如果你之前直接引用了 v2 包的类（如 TextStyleResolver），需要更新 import 路径。
